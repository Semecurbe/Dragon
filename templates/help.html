❓ Help

Dragon is a local RAG (Retrieval-Augmented Generation) application. You add your documents, and then ask questions about them in plain language — Dragon finds the relevant passages and uses an AI to compose an answer.

🚀 Quick start

  1. Go to ⚙️ Settings and enter your API key (or configure Ollama for a fully local setup).
  2. Go to 📤 Ingest and drop a PDF, DOCX, or PPTX file.
  3. Wait for the pipeline to finish — the document is now indexed.
  4. Go to 💬 Chat and ask a question.

📑 Tab by tab

Type your question and press Enter or click Send.

Dragon runs this pipeline for each question:

  1. Translation — your question is translated to English if needed, to improve semantic search quality.
  2. Vector search — the most relevant chunks from your documents are retrieved from ChromaDB.
  3. Context expansion — neighboring chunks (before and after each match) are added for surrounding context.
  4. LLM answer — the AI composes an answer grounded in the retrieved passages.

Click the ▶ Reasoning arrow below the answer to inspect which chunks were used and every step of the pipeline.

Filter — use the 🔍 Filter dropdown to restrict the search to one or more specific documents.

Attach a file — expand 📎 Add a document to the conversation to paste a one-off document directly into the context (it is not indexed).

Export — click 📥 .qmd to download the conversation as a Quarto Markdown file.

Browse the rendered HTML site generated by Quarto from your ingested documents.

Each document becomes a page with full formatting, images, and a navigation sidebar. An AI-generated executive summary is available at the top of each page.

The site is served locally on port 8080 and is also accessible from other devices on your network.

View and edit the raw .qmd (Quarto Markdown) source files that were generated from your documents during ingestion.

Each file is the intermediate representation used to build the Documentation site. You can correct conversion errors or add content manually, then re-render from the Documentation tab.

You can also delete a document here — this removes it from the vector database and from the documentation site.

Add a new document to Dragon. Drag and drop a file onto the zone, or click to browse.

Supported formats: PDF, DOCX, PPTX, HTML, Markdown (.md), AsciiDoc (.adoc), Excel (.xlsx).

The pipeline runs 7 steps — you can follow them in real time:

  1. Conversion to Markdown via Docling
  2. Image export
  3. Quarto page written
  4. Hierarchical chunking
  5. Vectors indexed in ChromaDB
  6. Quarto index rebuilt
  7. HTML site rendered

For batch ingestion of a whole folder, use the CLI tool: python3 ingest_dir.py /path/to/folder/

Split a large PDF into one file per chapter, based on its bookmark outline.

  1. Drop a PDF that has a table of contents / bookmarks.
  2. Dragon reads the top-level chapters and shows you the list with page ranges.
  3. Click Download ZIP — you receive a .zip containing one PDF per chapter, named 01_Chapter_title.pdf, 02_…, etc.

Tip: if your PDF has no bookmarks, add them first with a tool like PDFtk or Adobe Acrobat.

This feature uses PyMuPDF. If it is not installed, run pip install PyMuPDF and restart the app.

LLM provider — choose between Claude (Anthropic), Gemini (Google), ChatGPT (OpenAI), or Ollama (fully local). API keys are stored in .rag_config.json and never leave your machine.

System prompt — instructions given to the AI before every conversation. Customize the assistant's tone, language, or domain focus here.

Chunks retrieved — how many passages are pulled from the vector database per query (1–20). More chunks means broader context but more tokens consumed.

Neighbor chunks — for each matched chunk, also include the N chunks immediately before and after it in the source document. This gives the LLM more surrounding context. Set to 0 to disable.

💡 Tips

  • Questions can be in any language — Dragon auto-translates before searching.
  • Use the Filter dropdown in Chat to search only within a specific document.
  • The app is also reachable from other devices on your local network at the IP shown in the terminal.
  • Install Dragon as a PWA (click 📲 Install in the top bar) for a native-app feel.
  • For large document collections, increase Chunks retrieved in Settings and use precise filter keywords.
  • The Reasoning block (▶ arrow under each answer) shows exactly which chunks were used — useful for verifying sources.
